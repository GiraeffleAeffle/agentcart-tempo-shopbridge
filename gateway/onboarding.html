AgentCart Onboarding Flow

The demo needs two onboarding stories: a household user gives an agent spending authority under policy, and a WooCommerce shop opts in by exposing products, quotes, payment rails, order status, and refund metadata.

Dashboard Presentation Payments Field Map Two-Shop Demo
Household User Onboarding
  1. Install or connect an agent that can call AgentCart/ShopBridge APIs.
  2. Add delivery address, preferred merchants, blocked categories, and budgets.
  3. Connect an approval channel: chat, Home Assistant, Telegram, Signal, or another signed decision API.
  4. Choose payment credential source: Tempo MPP wallet/account or Stripe/card SPT flow.
  5. Require explicit approval for final quote, delivery window, merchant, and rail before checkout.
WooCommerce Shop Onboarding
  1. Install WooCommerce and configure products, stock, tax, shipping countries, terms, and returns.
  2. Install AgentCart ShopBridge.
  3. Configure support email, Tempo recipient, optional Stripe profile/network id, verifier URL, and merchant token.
  4. Publish /.well-known/agentcart.json and optionally register its hash.
  5. Run sandbox quote, paid order, status, and refund checks.

Buyer Journey

1. IntentUser says: buy my favorite tea.
2. DiscoveryAgent finds opt-in merchants and quote endpoints.
3. QuoteAgent gets final EUR totals, VAT, shipping, ETA, refund policy.
4. ApprovalUser approves exact quote and payment rail.
5. OrderPayment proof creates Woo order, task, calendar event, and audit record.

Refund Flow

Demo Woo refund record

AgentCart calls ShopBridge, which creates a WooCommerce refund record with real_refund_verified=false. This proves order-system integration only.

Stripe/card Card refund

The verifier calls Stripe refund APIs, returns a refund reference, then ShopBridge records the Woo refund and audit metadata.

Tempo Stablecoin refund

The verifier creates or verifies a rail-specific refund transfer back to the source wallet, then returns the refund transaction reference.

Live Demo Script

  1. Open the WooCommerce admin AgentCart settings page and show manifest, catalog, quote, payment verifier, Tempo recipient, and Stripe profile readiness.
  2. Open the two-shop quote tournament and show Woo wins on final price.
  3. Ask the household chat to buy Hazel's Chocolate Tea, approve the quote, and open the generated order proof page.
  4. Open the Woo order in admin and show the paid order line items.
  5. On the AgentCart proof page, click "Record Demo Refund" and show the Woo refund record plus AgentCart audit row.

Do not say the demo refund moved money. Say production uses the same endpoint shape but requires Stripe or Tempo verifier confirmation first.
