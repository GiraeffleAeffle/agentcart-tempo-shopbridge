Modular ShopBridge Stack

How WordPress, WooCommerce, AgentCart ShopBridge, and Tempo MPP fit together without turning the plugin into a payment processor.

Demo Cockpit Deck Quote Tournament
WordPress

Site shell

Admin UI, users, plugin system, pages, hosting, updates.

  • Store owner already knows this
  • Plugin installs like normal WordPress software
WooCommerce

Commerce source of truth

Products, stock, VAT/tax, shipping, orders, refunds, and fulfillment.

  • Merchant remains merchant of record
  • Normal order admin still works
ShopBridge

Agent-readable adapter

Machine-readable manifest, catalog, quote, paid-order, status, and refund endpoints.

  • No scraping or browser automation
  • Quote hash binds final terms
AgentCart

Buyer-side safety

Discovery, quote comparison, household policy, approval, audit, delivery calendar, and task sync.

  • Approval before checkout
  • Chooses by final quote, not ads
Tempo MPP

Payment proof rail

HTTP 402-style payment challenge, payment authorization, and receipt/proof.

  • Demo uses testnet proof
  • Production needs verifier-backed settlement
Operations

Production controls

FX, disputes, refunds, carrier tracking, KYC/payouts, terms, privacy, and consumer rights.

  • Not hidden by the prototype
  • Handled by merchant + PSP/verifier

What Tempo MPP does

Payment challengeWWW-Authenticate: Payment / HTTP 402-shaped request.
Payment credentialAgent presents an MPP-compatible authorization.
Receipt/proofAgentCart stores the receipt and explorer/proof reference when available.

What ShopBridge adds around it

Before paymentProduct, stock, final price, VAT, shipping, delivery estimate, merchant of record, quote hash.
After paymentWooCommerce paid order, order status URL, refund metadata, delivery/status handoff.
SafetyHousehold budget, category policy, explicit approval, idempotency, replay protection, audit log.

Merchant setup

Install the plugin through WordPress, configure support contact, shipping countries, payment verifier/recipient, then choose or bulk-enable which WooCommerce products are AgentCart-enabled. The manifest advertises endpoints; the catalog endpoint exposes only those opted-in products.

Agent flow

Intent -> merchant discovery -> private quote tournament -> policy check -> approval -> MPP proof -> Woo order -> audit and delivery ETA.

Why modular

WooCommerce keeps merchant operations. Tempo keeps payment semantics. AgentCart connects agents to both without requiring a new storefront.

Local demo boundary: the bundled demo shows a real WooCommerce order surface and Tempo testnet proof. It does not claim real EUR settlement, carrier tracking, or production refunds unless a payment verifier and merchant operations are configured.
