AgentCart

A WooCommerce ShopBridge that lets household agents safely buy from opt-in merchants with Tempo MPP payment proof.

MPP can answer "can an agent pay?" AgentCart answers the missing merchant question: how does a normal shop expose products, final quotes, approval, orders, delivery, refunds, and audit data to agents?
Problem

Payment is only one part of checkout.

Tempo MPP

Gives machine-readable payment challenges, authorization, and receipts.

Small shops

Need catalog, VAT, stock, shipping, merchant-of-record, refund, and fulfillment data exposed safely.

Household agents

Need budgets, category rules, explicit consent, delivery visibility, and an audit trail.

ShopBridge Plugin

A normal WooCommerce shop can opt in without scraping.

1. InstallUpload/activate AgentCart ShopBridge like a normal WordPress plugin.
2. Add productsUse standard WooCommerce products, stock, prices, images, tax, and shipping.
3. Expose/.well-known/agentcart.json advertises endpoints; /catalog lists published products.
4. Pay proofAgentCart completes MPP-shaped checkout and attaches Tempo testnet proof.
5. OrderWooCommerce receives a paid Processing order the merchant can fulfill.

In the demo: Plugins -> AgentCart ShopBridge, Products -> four Woo products, Manifest/Catalog JSON -> agent-readable product feed.

Boundaries

Practical bridge, not agent magic.

No scraping

Only opt-in merchants. No browser automation against real shops.

No silent spend

Budget, category policy, idempotency, replay protection, and explicit approval happen before checkout.

No fake EUR settlement claim

The demo uses EUR quotes plus Tempo testnet/pathUSD proof. Production needs verifier-backed settlement and refunds.
