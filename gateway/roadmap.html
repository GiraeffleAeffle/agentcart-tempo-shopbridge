AgentCart Production Roadmap

A working map from hackathon prototype to production-grade agent commerce for WooCommerce merchants and buyer agents. Percentages are weighted by production readiness, not by demo coverage.

Overall Readiness

47%

The prototype has the right shape: WooCommerce plugin, manifest/catalog/quote/order endpoints, buyer approval, payment proof attachment, registry groundwork, protocol profiles, x402 shim, signed request gate, and local skill path. The remaining work is mostly production hardening, real settlement operations, standards adapters, and merchant onboarding polish.

Area Breakdown

Next Slice

  • Merchant registry transparency and refresh UX: verified domain proof, catalog hash freshness, revoke state, and safer automatic refresh.
  • Payment verifier hardening: quote-bound settlement, durable replay store, provider metrics, and clearer failure states.
  • WooCommerce production gate: Plugin Check, PHPCS, packaging, uninstall behavior, and integration smoke coverage.

Highest Risks

  • Payment rails can be demo-valid while still not representing real EUR settlement or provider-grade refunds.
  • Merchant discovery needs authenticity, freshness, and prompt-injection resistant data handling before broad indexing.
  • Buyer-side adoption depends on a skill-only path or very small local runtime, not a heavy separate service.

Standards Direction

  • Keep AgentCart as the commerce safety layer around payment standards, not as a replacement for them.
  • Expose translators for x402, MCP tools, AP2-style mandates, UCP-style checkout data, and future registry proofs.
  • Prefer canonical JSON plus optional compact agent formats only after the contract is stable.

Gate Overview

Release Phases

Alpha

Local buyer agents and trusted pilot shops

Focus on agent-readable WooCommerce, explicit approval, quote binding, registry transparency, and repeatable local setup. This is mostly implemented.

Beta

Real merchant operations

Add real settlement/refund workflows, shipping/tracking, observability, plugin security review, merchant setup wizard, and stable WordPress packaging.

Public

Discoverable agent commerce network

Launch registry governance, standards translators, abuse controls, docs, WordPress.org listing, and verified pilot merchants that agents can compare safely.

Definition Of Done

AreaProduction barCurrent gap
Merchant setup A Woo merchant installs the ZIP or WordPress.org plugin, enables AgentCart, picks payment rails, and publishes a valid manifest without touching code. Quick Start and readiness UX exist. Test checkout automation, hosted registry connection, and Plugin Check hardening remain.
Buyer setup A buyer agent can use a small skill/tool package to discover merchants, compare final quotes, ask for approval, pay, and track orders. Skill-only path exists as an alpha. It needs packaging, docs, and more agent runtime adapters.
Trust Agents can verify merchant identity, endpoint freshness, signed sensitive calls, and quote/order integrity without trusting arbitrary page text. Registry and signed request gates are present. Freshness automation, revocation UX, and stronger registry backing remain.
Payments Payment proof is quote-bound, amount-correct, replay-safe, provider-verifiable, refundable, and auditable across settlement rails. Demo rails and verifier contracts are implemented. Real provider operations and production replay/metrics remain.
Operations Release, monitoring, logs, failure modes, support docs, and plugin review gates are boring and repeatable. Verify pipeline exists. WordPress.org readiness, monitoring, and support runbooks still need work.
